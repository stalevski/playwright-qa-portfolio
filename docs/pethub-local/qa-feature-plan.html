Design principles applied

  • Additive only. Existing v1 endpoints and their tests are untouched; new behaviour lives under /api/v2, /api/auth, /api/jobs and observability paths.
  • Deterministic by construction. Signed tokens use a fixed secret; rate-limit buckets are keyed by a client-supplied X-Client-Id; async jobs advance by poll count, not wall-clock; idempotency keys persist for the process lifetime.
  • One layer per concern. Pure logic in platform.ts, HTTP wiring in qa.routes.ts, typed transport in platform.dto.ts, a dedicated client returning raw responses for status-code assertions.
